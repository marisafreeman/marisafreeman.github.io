1. UFW 简介

1.1 什么是 UFW

UFW（Uncomplicated Firewall）是 Ubuntu 和其他基于 Debian 的 Linux 发行版上的默认防火墙管理工具。UFW 的目标是简化防火墙的管理，尤其是对于初学者来说，使其能够轻松地管理和配置防火墙规则。

在 Linux 中，防火墙主要通过 iptables 来管理。UFW 其实是对 iptables 的封装，它通过简单的命令行接口提供易于理解和使用的防火墙管理功能。UFW 使得用户可以快速启用和配置防火墙规则，而无需深入了解复杂的 iptables 语法。

1.2 为什么选择 UFW

选择 UFW 的原因通常有以下几个：

  • 简单易用：UFW 提供了一个简化的命令行界面，使得防火墙规则的管理变得更加直观，适合初学者。
  • 默认集成：UFW 是 Ubuntu 默认安装的防火墙工具，适合大多数 Ubuntu 用户。
  • 强大且灵活：尽管 UFW 的配置相对简单，但它仍然提供了强大的功能，足以满足大部分家庭和小型服务器的防火墙需求。
  • 易于集成：UFW 可以与其他安全工具（如 Fail2ban）配合使用，以增强防护能力。

1.3 UFW 和其他防火墙管理工具对比

尽管 UFW 是 Ubuntu 上最常用的防火墙工具，但它并不是唯一的选择。以下是 UFW 与其他几种常见防火墙管理工具的对比：

  • iptables：iptables 是 Linux 系统上最底层的防火墙管理工具，它提供了最强大、最灵活的功能，但也需要较高的技术水平才能正确配置。相比之下，UFW 是一个更高级的封装工具，简化了配置过程。

  • Firewalld：Firewalld 是一种动态管理防火墙的工具，通常在 CentOS 或 RHEL 系统中使用。它提供了比 UFW 更复杂的区域化管理和动态规则调整功能。但对于大多数 Ubuntu 用户来说，UFW 更加简便。

  • CSF (ConfigServer Security & Firewall)：CSF 是一种专为 cPanel 用户设计的防火墙管理工具，通常用于托管服务器的管理面板中。它集成了多种功能，适合有复杂需求的用户。

尽管有其他工具可供选择，UFW 因其简洁性和易用性，通常是小型服务器和个人计算机的首选。

3. 基本命令与操作

在这一章中，我们将介绍如何使用 UFW 执行基本的防火墙操作。你将学习如何允许或拒绝特定端口、设置默认策略以及查看和删除防火墙规则。

    sudo ufw enable

  • 禁用防火墙：

sudo ufw status verbose

3.3 允许和拒绝端口

在防火墙中，您可以指定允许或拒绝哪些端口上的流量。以下是常用的命令：

  • 允许某个端口的流量：

    例如，允许来自任意地方的 HTTP 流量（端口 80）：

    bash
    sudo ufw allow 80

    您也可以指定端口和协议类型（TCP 或 UDP）：

    bash
    sudo ufw allow 80/tcp

    允许所有访问端口 22（SSH）：

    bash
    sudo ufw allow ssh

    对于常见的服务，UFW 提供了预定义的应用规则，您可以直接使用这些规则名称，而无需指定端口号。

  • 拒绝某个端口的流量：

    例如，拒绝来自任意地方的 8080 端口的流量：

    bash
    sudo ufw deny 8080

    或者拒绝特定的协议（例如 UDP）：

    bash
    sudo ufw deny 53/udp

  • 删除规则：

    如果需要删除某个规则，可以使用以下命令：

    bash
    sudo ufw delete allow 80

    该命令会删除允许端口 80 的规则。删除规则时，您需要指定要删除的规则内容，例如允许的端口或服务。

3.4 设置默认策略

UFW 提供了设置默认策略的功能，默认策略决定了在没有明确规则的情况下，防火墙如何处理流量。通常，您可以设置：

  • 默认拒绝所有入站流量，允许所有出站流量：

    这是一种非常常见的安全策略，即拒绝所有进入系统的流量，允许所有外出流量：

    bash
    sudo ufw default deny incoming
sudo ufw default allow outgoing

    这样设置后，任何没有明确允许的入站连接都会被拒绝，而出站连接则不受限制。

  • 修改默认策略：

    如果您想更改默认策略（例如，允许所有入站流量），可以使用以下命令：

    bash
    sudo ufw default allow incoming
sudo ufw default deny outgoing

    请谨慎更改默认策略，避免造成系统安全隐患。

3.5 允许特定 IP 地址或子网

除了允许端口外，您还可以允许特定的 IP 地址或整个子网访问某些服务。例如，允许来自 192.168.1.100 的 IP 地址访问 SSH（端口 22）：

bash
sudo ufw allow from 192.168.1.100 to any port 22

允许整个子网（例如，192.168.1.0/24）访问端口 80（HTTP）：

bash
sudo ufw allow from 192.168.1.0/24 to any port 80

    bash
    sudo ufw delete 1

    这样会删除允许端口 22 的规则。

11.2.1 参考命令

以下是一些常用的 UFW 命令，帮助您快速管理防火墙：

  • 查看防火墙状态：

    bash
    sudo ufw status
sudo ufw status verbose

  • 启用和禁用防火墙：

    bash
    sudo ufw enable
sudo ufw disable

  • 允许或拒绝特定端口：

    bash
    sudo ufw allow <port>/tcp  # 允许 TCP 端口
sudo ufw deny <port>/tcp   # 拒绝 TCP 端口

  • 允许或拒绝特定 IP 地址的访问：

    bash
    sudo ufw allow from <IP address>
sudo ufw deny from <IP address>

  • 删除规则：

    bash
    sudo ufw delete <规则编号>

  • 启用日志记录：

    bash
    sudo ufw logging on
11.2.2 UFW 配置文件

UFW 配置文件存储在 /etc/ufw/ 目录中，您可以直接编辑这些文件来进行高级配置。以下是一些常见的配置文件：

  • /etc/ufw/ufw.conf： 包含防火墙的全局设置，如启用状态。

  • /etc/ufw/user.rules： 存储用户自定义的防火墙规则。

  • /etc/ufw/user6.rules： 存储与 IPv6 相关的防火墙规则。
